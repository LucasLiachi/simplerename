# ADR-001: Stack de Build e Distribuição Windows
**Date:** 2026-06-25
**Status:** Proposed

---

## Context

O projeto precisa gerar um executável Windows (.exe) instalável por usuários sem Python instalado.
Hoje existe um processo manual com scripts `.bat` e dois arquivos NSIS (`installer.nsi` e
`windows_installer.nsi`) de origem indefinida. Precisamos de um processo automatizado, reproduzível
e versionado que rode em CI/CD.

As opções avaliadas são: PyInstaller (ferramenta atual), Nuitka (compilação para C), e cx_Freeze.
Para o empacotamento em installer: NSIS (atual), Inno Setup, e WiX Toolset.

## Decision

Manter **PyInstaller** para geração do `.exe` e **NSIS** para o installer, ambos orquestrados por
**GitHub Actions** com runner `windows-latest`. A versão é controlada via `src/version.py` (única
fonte de verdade) e injetada no build pela tag git.

## Alternatives Considered

| Opção | Prós | Contras | Razão Rejeitada |
|---|---|---|---|
| **PyInstaller** ✓ | Já usado no projeto; suporte nativo a PyQt6; grande comunidade | Executável maior (~50MB); mais lento para iniciar | — (escolhido) |
| Nuitka | Executável menor e mais rápido; compila para C nativo | Configuração complexa com PyQt6; build demorado no CI | Complexidade desnecessária para o estágio atual |
| cx_Freeze | Leve, simples | Suporte a PyQt6 menos maduro; menos documentação | Risco técnico desnecessário |
| **NSIS** ✓ | Já presente no projeto; amplamente suportado no GitHub Actions | Sintaxe verbosa | — (escolhido) |
| Inno Setup | Sintaxe mais moderna; GUI para configurar | Menos familiaridade; Actions action menos madura | Curva de aprendizado sem benefício claro |
| WiX Toolset | Padrão Microsoft; suporta MSI | Muito complexo; overengineering para este projeto | Overkill |

## Consequences

**Positivo:**
- Processo de release totalmente automatizado: `git tag` → instalador publicado no GitHub Releases
- Desenvolvedor não precisa de ambiente Windows para gerar builds (CI usa `windows-latest`)
- Usuário final instala/atualiza com duplo-clique, sem instalar Python

**Negativo:**
- Executável gerado pelo PyInstaller tem ~50-80MB (PyQt6 é grande)
- Cold start do `.exe` pode levar 2-4 segundos na primeira execução (PyInstaller descompacta para temp)

**Riscos:**
- Antivírus Windows pode bloquear executáveis PyInstaller gerados por CI sem code signing
- Mitigação: documentar para o usuário ou adicionar code signing via certificado em versão futura
