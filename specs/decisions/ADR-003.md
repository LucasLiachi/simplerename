# ADR-003 — CI/CD: Decisões de Build e DevOps

**Status:** Aceito  
**Data:** 2026-06-26  
**Autor:** PP-Planner / Lucas Liachi

---

## Contexto

Durante a implementação de FEATURE-001 (Build Pipeline) e os ciclos de correção subsequentes
(v1.0.0 → v1.0.4), o pipeline de CI/CD no GitHub Actions apresentou uma série de falhas que
revelaram incompatibilidades entre as suposições da spec original e o ambiente real de execução
(`windows-latest`). Este ADR documenta cada decisão tomada e a razão por trás dela.

---

## Decisões

### 1. Usar `shell: bash` para o step do PyInstaller

**Problema:** O comando `pyinstaller ... ^` usa `^` como continuação de linha do CMD. O runner
`windows-latest` executa steps `run:` com PowerShell (`pwsh.exe`) por padrão. PowerShell não
reconhece `^` como continuação — interpreta `--` como operador unário e falha com `ParserError`.

**Decisão:** Adicionar `shell: bash` ao step do PyInstaller e trocar `^` por `\` (bash).

```yaml
- name: Build executable with PyInstaller
  shell: bash
  run: |
    pyinstaller --onefile --windowed \
      --icon=resources/icons/simplerename.ico \
      --name=SimpleRename \
      --add-data="resources;resources" \
      main.py
```

**Alternativa rejeitada:** Usar backtick (`` ` ``) do PowerShell — funciona, mas `shell: bash`
é mais portátil e legível.

---

### 2. Instalar Pillow no CI para conversão do ícone

**Problema:** O arquivo `resources/icons/simplerename.ico` não era um ICO válido (provavelmente
PNG renomeado). PyInstaller tenta converter via Pillow, mas Pillow não estava instalado.

**Decisão:** Adicionar `pillow` ao `pip install` do CI e gerar um ICO válido via Python antes
do PyInstaller:

```yaml
- name: Install dependencies
  run: pip install -r requirements.txt pillow

- name: Generate valid ICO
  shell: python
  run: |
    from PIL import Image, ImageDraw
    img = Image.new("RGBA", (256, 256), (33, 150, 243, 255))
    # ... desenha ícone simples
    img.save("resources/icons/simplerename.ico", format="ICO",
             sizes=[(256,256),(128,128),(64,64),(48,48),(32,32),(16,16)])
```

**Razão para não adicionar Pillow ao `requirements.txt`:** Pillow é necessário apenas para o
build do CI, não para execução da aplicação. Manter `requirements.txt` enxuto.

---

### 3. Instalar NSIS via Chocolatey

**Problema:** `joncloud/makensis-action@v4` foi usado inicialmente, mas a action não instala o
NSIS — apenas o encontra no PATH. O NSIS não vem pré-instalado no runner `windows-latest`.

**Decisão:** Instalar NSIS explicitamente e chamar `makensis.exe` diretamente:

```yaml
- name: Install NSIS
  run: choco install nsis --no-progress -y

- name: Build NSIS Installer
  run: |
    & "C:\Program Files (x86)\NSIS\makensis.exe" /DVERSION=${{ steps.version.outputs.VERSION }} installer.nsi
```

**Razão para abandonar `joncloud/makensis-action`:** Além de não instalar o NSIS, a action
usava Node 20 (deprecado), adicionando warnings ao log. Chamada direta é mais simples.

---

### 4. Corrigir caminhos no `installer.nsi`

**Problemas encontrados:**

| Linha original | Problema | Correção |
|---|---|---|
| `OutFile "dist-windows/SimpleRename-Setup-${VERSION}.exe"` | Pasta `dist-windows/` não existe no CI | `OutFile "SimpleRename-Setup-${VERSION}.exe"` (raiz) |
| `File "dist-windows\SimpleRename.exe"` | PyInstaller gera em `dist/`, não `dist-windows/` | `File "dist\SimpleRename.exe"` |
| `File /r "resources\icons\..." "$INSTDIR\..."` | Sintaxe NSIS inválida (File não aceita destino como segundo arg) | `SetOutPath "$INSTDIR\resources\icons"` + `File "resources\icons\..."` |

---

### 5. Pinar versões de GitHub Actions a releases compatíveis com Node 24

**Problema:** A partir de junho de 2026, o runner `windows-latest` passou a usar Node 24 por
padrão. Actions que apontam para `@v4` (sem patch) podem estar usando versões internas com Node 20,
gerando warnings de deprecação.

**Decisão:** Pinar todas as actions a versões específicas já migradas para Node 24:

```yaml
uses: actions/checkout@v4.2.2          # Node 24 nativo
uses: actions/setup-python@v5.6.0      # Node 24 nativo
uses: softprops/action-gh-release@v2.3.2  # Node 24 nativo
```

**Critério de atualização:** Verificar o changelog da action quando aparecer warning de Node
deprecado. Pinar sempre ao patch mais recente da major usada.

---

### 6. Correção em runtime: `Qt.ItemFlag` vs `Qt.ItemFlags`

**Problema (pós-build):** A aplicação compilada pelo PyInstaller exibia erro ao iniciar:
```
AttributeError: type object 'Qt' has no attribute 'ItemFlags'
```

O método `flags()` do `DualBandTableModel` usava `Qt.ItemFlags` (plural) como tipo de retorno,
que não existe em PyQt6. Existe apenas `Qt.ItemFlag` (singular).

**Decisão:** Corrigir o type annotation e nunca usar `Qt.ItemFlags` em PyQt6:

```python
# ERRADO (PyQt5)
def flags(self, index) -> Qt.ItemFlags:

# CORRETO (PyQt6)
def flags(self, index) -> Qt.ItemFlag:
```

**Nota:** Os valores usados no corpo (`Qt.ItemFlag.ItemIsEnabled`, etc.) já estavam corretos —
o erro era apenas no tipo de retorno da assinatura.

---

## Consequências

- O pipeline produz builds reproduzíveis: qualquer `git tag vX.Y.Z && git push origin vX.Y.Z`
  gera o instalador sem intervenção manual.
- Correções de CI usam patch bump (v1.0.0 → v1.0.1) para preservar o histórico de falhas
  e facilitar diagnóstico retroativo.
- O arquivo `installer.nsi` é definitivo e está commitado — `windows_installer.nsi` foi removido.
- Pillow não entra em `requirements.txt` para não aumentar o tamanho do executável.
