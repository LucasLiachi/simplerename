### Passo 1 de 4 — Corrigir `_parse_filename()` · `src/search_pipeline.py`

**Por que primeiro:** sem o parser correto, as Estratégias 3 e 4 do `SearchPipeline` nunca
geram autor, e o `MetadataLookupService` busca só por título — retornando resultados errados
ou nenhum. Todos os demais bugs dependem de a busca ter dados corretos para propagar.

**O que fazer:**

Reescrever `FILENAME_PATTERNS` com a seguinte ordem de prioridade:

```python
FILENAME_PATTERNS = [
    # 1. ISBN no início do nome
    (r'^(?P<isbn>97[89]\d{10})\s*[-_\s]\s*(?P<title>.+)$',
     ["isbn", "title"]),

    # 2. Título - Autor (Ano)   ← padrão mais comum na biblioteca
    (r'^(?P<title>.+?)\s+-\s+(?P<author>[^()\d][^()]{2,}?)(?:\s+\((?P<year>\d{4})\))?$',
     ["title", "author", "year"]),

    # 3. SOBRENOME, Nome - Título (Ano)   ← convenção ABNT
    (r'^(?P<last>[A-ZÀ-Ý][A-ZÀ-Ýa-zà-ý]+),\s+(?P<first>[^-]{2,}?)'
     r'\s+-\s+(?P<title>.+?)(?:\s+\((?P<year>\d{4})\))?$',
     ["author_last_first", "title", "year"]),

    # 4. Título (Ano)
    (r'^(?P<title>.+?)\s+\((?P<year>\d{4})\)$',
     ["title", "year"]),

    # 5. Fallback
    (r'^(?P<title>.+)$', ["title"]),
]
```

Adicionar em `tests/test_search_pipeline.py` os seguintes 6 casos parametrizados:

```python
@pytest.mark.parametrize("stem,expected", [
    ("O Estrangeiro - Albert Camus",
     {"title": "O Estrangeiro", "author": "Albert Camus"}),
    ("O Poder Do Agora - Eckhart Tolle",
     {"title": "O Poder Do Agora", "author": "Eckhart Tolle"}),
    ("REED, John - 10 dias que abalaram o mundo (2006)",
     {"author": "REED, John", "title": "10 dias que abalaram o mundo", "year": "2006"}),
    ("A Caminho da Luz - (Emmanuel) Francisco Candido Xavier",
     {"title": "A Caminho da Luz", "author": "(Emmanuel) Francisco Candido Xavier"}),
    ("9788535902778 - Dom Casmurro",
     {"isbn": "9788535902778", "title": "Dom Casmurro"}),
    ("O Poder do Agora (2002)",
     {"title": "O Poder do Agora", "year": "2002"}),
])
def test_parse_filename(stem, expected):
    result = _parse_filename(stem)
    for key, value in expected.items():
        assert result.get(key) == value, f"{stem!r}: esperava {key}={value!r}, obteve {result}"
```

**Critério de aceite:** todos os 6 testes passam com `pytest tests/test_search_pipeline.py -v`.

### Passo 2 de 4 — Corrigir propagação do resultado da busca · `src/file_manager.py` + `src/main_window.py`

**Por que segundo:** com o parser gerando dados corretos, a busca encontra resultados — mas
`_on_search_row_done()` não emite `dataChanged` com os roles corretos, então a view não
redesenha as células da faixa verde.

**O que fazer em `src/file_manager.py` (DualBandTableModel):**

Adicionar o método:

```python
def update_row(self, row_idx: int, new_row: FileRow) -> None:
    """Substitui FileRow na posição e força redesenho da linha inteira."""
    if 0 <= row_idx < len(self.rows):
        self.rows[row_idx] = new_row
        tl = self.index(row_idx, 0)
        br = self.index(row_idx, self.columnCount(None) - 1)
        self.dataChanged.emit(
            tl, br,
            [Qt.ItemDataRole.DisplayRole,
             Qt.ItemDataRole.BackgroundRole,
             Qt.ItemDataRole.DecorationRole]
        )
```

**O que fazer em `src/main_window.py`:**

Substituir o corpo de `_on_search_row_done()` por:

```python
def _on_search_row_done(self, row_idx: int, updated_row: object) -> None:
    self.model.update_row(row_idx, updated_row)
    self.spreadsheet_view.viewport().update()
```

**Critério de aceite:** após clicar "Buscar (Linha)" numa linha com nome `"O Estrangeiro -
Albert Camus.epub"`, os campos Novo Título, Novo Autor, Novo Ano e Nova Editora da faixa
verde aparecem preenchidos com fundo âmbar em ≤ 8 segundos.

### Passo 3 de 4 — Corrigir cores em dark mode · `src/spreadsheet_view.py`

**Por que terceiro:** com dados chegando na view, os fundos de célula ficam visíveis — mas
as cores hardcoded `#E6F1FB` e `#EAF3DE` são invisíveis no dark mode do Windows 11.

**O que fazer:**

Substituir toda lógica de cor de célula no `DualBandTableModel.data()` por uma função
auxiliar `_cell_background()` com detecção de tema:

```python
def _cell_background(self, col: int, row_data: FileRow) -> QColor:
    is_dark = (QApplication.palette()
               .color(QPalette.ColorRole.Window)
               .lightness() < 128)

    if col in BLUE_COLS:
        return QColor(30, 60, 100) if is_dark else QColor(210, 230, 248)

    if col == PREVIEW_COL:
        return QColor(50, 50, 60) if is_dark else QColor(235, 235, 235)

    # Faixa verde — cor por estado
    field_key = GREEN_COL_KEYS.get(col)
    if not field_key:
        return QColor()
    value     = getattr(row_data, field_key, None)
    confirmed = row_data.field_confirmed.get(field_key, False)

    if confirmed and value:
        return QColor(20, 70, 30)   if is_dark else QColor(200, 235, 200)  # verde
    if value is not None:
        return QColor(80, 55, 10)   if is_dark else QColor(255, 235, 180)  # âmbar
    return QColor(45, 45, 50)       if is_dark else QColor(255, 255, 255)  # vazio
```

Aplicar o mesmo padrão no `GroupedHeaderView.paintSection()` para as cores de cabeçalho:

```python
is_dark = self.palette().color(QPalette.ColorRole.Window).lightness() < 128
GROUP_BG = {
    "Estado Atual":        QColor(25, 55, 100)  if is_dark else QColor(181, 212, 244),
    "Proposta de Mudança": QColor(20, 65, 30)   if is_dark else QColor(159, 225, 203),
}
GROUP_FG = {
    "Estado Atual":        QColor(180, 210, 255) if is_dark else QColor(12, 68, 124),
    "Proposta de Mudança": QColor(160, 230, 180) if is_dark else QColor(8, 80, 65),
}
```

**Critério de aceite:** faixas azul e verde visualmente distintas com contraste legível
tanto em dark mode quanto em light mode; sem necessidade de reiniciar o app ao trocar tema.

### Passo 4 de 4 — Refatorar toolbar · `src/main_window.py`

**Por que por último:** é limpeza visual; não desbloqueia nem compromete nada funcional.
Pode ser feito a qualquer momento após os passos 1-3, mas convém fazer em sequência.

**O que fazer:**

Remover completamente as `QAction` de "Prepare Rename" e "Replace Spaces" (definição +
conexão de sinal + entrada na toolbar).

Substituir `_setup_toolbar()` pela versão abaixo — 11 botões, 5 grupos, labels em
português, tooltips e estados habilitado/desabilitado:

```python
def _setup_toolbar(self) -> None:
    tb = self.addToolBar("Principal")
    tb.setMovable(False)

    # Grupo 1 — Identificar
    self.browse_action = QAction("Abrir Pasta", self)
    self.browse_action.setToolTip("Selecionar a pasta com os arquivos a organizar")
    tb.addAction(self.browse_action)
    tb.addSeparator()

    # Grupo 2 — Buscar
    self.search_selected_action = QAction("Buscar (Linha)", self)
    self.search_selected_action.setToolTip("Buscar metadados online para a linha selecionada")
    self.search_selected_action.setEnabled(False)

    self.search_incomplete_action = QAction("Buscar Incompletos", self)
    self.search_incomplete_action.setToolTip("Buscar metadados para linhas sem dados (🔴/🟡)")
    self.search_incomplete_action.setEnabled(False)

    self.search_all_action = QAction("Buscar Todos", self)
    self.search_all_action.setToolTip("Buscar metadados para todas as linhas da planilha")
    self.search_all_action.setEnabled(False)

    tb.addAction(self.search_selected_action)
    tb.addAction(self.search_incomplete_action)
    tb.addAction(self.search_all_action)
    tb.addSeparator()

    # Grupo 3 — Revisar
    self.confirm_row_action = QAction("Confirmar Linha ✓", self)
    self.confirm_row_action.setToolTip("Aceitar todas as sugestões da linha selecionada")
    self.confirm_row_action.setEnabled(False)

    self.confirm_all_action = QAction("Confirmar Todos ✓✓", self)
    self.confirm_all_action.setToolTip("Aceitar todas as sugestões de todas as linhas")
    self.confirm_all_action.setEnabled(False)

    self.clear_proposal_action = QAction("Limpar Proposta ✗", self)
    self.clear_proposal_action.setToolTip("Apagar a faixa verde da linha selecionada")
    self.clear_proposal_action.setEnabled(False)

    tb.addAction(self.confirm_row_action)
    tb.addAction(self.confirm_all_action)
    tb.addAction(self.clear_proposal_action)
    tb.addSeparator()

    # Grupo 4 — Aplicar
    self.apply_action = QAction("Aplicar Rename ▶", self)
    self.apply_action.setToolTip("Renomear arquivos e gravar metadados confirmados no PDF")
    self.apply_action.setEnabled(False)

    self.apply_folders_action = QAction("Aplicar com Pastas 📁", self)
    self.apply_folders_action.setToolTip("Renomear e mover para subpastas CDD")
    self.apply_folders_action.setEnabled(False)

    tb.addAction(self.apply_action)
    tb.addAction(self.apply_folders_action)
    tb.addSeparator()

    # Grupo 5 — Histórico
    self.undo_action = QAction("Desfazer ↩", self)
    self.undo_action.setToolTip("Reverter o último rename aplicado em disco")
    self.undo_action.setEnabled(False)

    self.redo_action = QAction("Refazer ↪", self)
    self.redo_action.setToolTip("Refazer o rename desfeito")
    self.redo_action.setEnabled(False)

    tb.addAction(self.undo_action)
    tb.addAction(self.redo_action)

    # Conectar sinais (substituir conexões antigas)
    self.browse_action.triggered.connect(self._on_browse)
    self.search_selected_action.triggered.connect(self._on_search_selected)
    self.search_incomplete_action.triggered.connect(self._on_search_incomplete)
    self.search_all_action.triggered.connect(self._on_search_all)
    self.confirm_row_action.triggered.connect(self._on_confirm_row)
    self.confirm_all_action.triggered.connect(self._on_confirm_all)
    self.clear_proposal_action.triggered.connect(self._on_clear_proposal)
    self.apply_action.triggered.connect(self._on_apply)
    self.apply_folders_action.triggered.connect(self._on_apply_folders)
    self.undo_action.triggered.connect(self._on_undo)
    self.redo_action.triggered.connect(self._on_redo)
```

Adicionar método `_update_toolbar_state()` chamado sempre que a seleção ou o estado do
model mudar, que habilita/desabilita cada botão conforme a tabela:

| Botão | Habilitado quando |
|---|---|
| Buscar (Linha) | `len(self.spreadsheet_view.selectedIndexes()) > 0` |
| Buscar Incompletos | `any(r.metadata_quality != MetadataQuality.COMPLETE for r in self.model.rows)` |
| Buscar Todos | `len(self.model.rows) > 0` |
| Confirmar Linha | Linha selecionada tem campo não confirmado na faixa verde |
| Confirmar Todos | `any(not all(r.field_confirmed.values()) for r in self.model.rows if r.new_title)` |
| Limpar Proposta | Linha selecionada tem qualquer dado na faixa verde |
| Aplicar Rename | `any(all(r.field_confirmed.get(k) for k in ["new_filename"]) for r in self.model.rows)` |
| Aplicar com Pastas | Mesmo critério de Aplicar Rename |
| Desfazer | `self.history_manager.canUndo()` |
| Refazer | `self.history_manager.canRedo()` |

**Critério de aceite:** toolbar com exatamente 11 botões em 5 grupos; "Prepare Rename" e
"Replace Spaces" inexistentes; cada botão habilitado/desabilitado conforme a tabela acima.

## Exit criteria global da FEATURE-008

Antes de criar a tag `v1.1.2`, verificar manualmente com a pasta `D:/4 - Biblioteca/Ajustando`:

1. Abrir pasta → faixa azul preenchida com os dados dos 4 arquivos; indicadores ⬤ corretos
2. Clicar "Buscar Incompletos" → faixa verde preenchida com âmbar para todos os arquivos
   cujo nome segue `Título - Autor` (inclui "O Estrangeiro - Albert Camus.epub")
3. Clicar "Confirmar Todos" → células ficam verdes
4. Clicar "Aplicar Rename ▶" → arquivos renomeados em disco; PDF com metadados gravados
5. Ctrl+Z → rename revertido em disco
6. Dark mode e light mode: faixas distinguíveis, texto legível
7. Toolbar tem 11 botões; "Prepare Rename" e "Replace Spaces" ausentes
